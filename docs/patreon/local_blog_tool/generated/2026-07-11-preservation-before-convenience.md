## Key Links

One of the design principles behind Biblion is preservation before convenience. That phrase is simple, but it shapes a large number of technical decisions.

Convenience-driven systems often optimize for speed, minimal friction, and the appearance of immediate results. That can be useful, but it becomes dangerous when working with historical texts and source materials that need to remain intelligible over time. Once context is stripped away, evidence is flattened, or provenance disappears, it becomes much harder to trust the output.

Preservation before convenience means the source should remain recoverable. It means transformations should be traceable. It means the workflow should help scholars and editors understand what changed, not simply present a final answer as if it emerged from nowhere.

This principle does not reject automation. It places automation in the right position. The machine helps with labor, pattern recognition, and acceleration, but it does not get to erase the documentary logic of the work.

That distinction matters for Biblion because the goal is not merely to process text. The goal is to steward materials responsibly while making them more usable for real scholarship. It is also one of the ideas now sitting underneath the public-facing home page and the broader story the project is starting to tell:
https://biblionocr.onrender.com/

If people are going to trust tools like this, they need to understand that the software is not being built only for convenience. It is being built to preserve evidence while making serious work more possible.

## Points to Emphasize

Convenience is useful, but it becomes dangerous when it hides provenance or flattens evidence.

Preservation-first design does not reject automation; it constrains automation so that evidence remains recoverable and interpretable.

This principle explains many downstream workflow and architecture decisions across the project.
